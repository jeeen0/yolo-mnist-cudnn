# live/ — 수업 실시간 카메라 데모

영상 클립 대신 **웹캠**으로 같은 과제-공지 파이프라인을 돌리는 시연용 버전입니다.
스코어가 매겨지는 파이프라인(`scripts/03_pipeline.py` + `preprocess.py` +
`mnistCUDNN`)을 **그대로 재사용**하고 프레임 소스만 카메라로 바꿉니다 — 같은 숫자에
대해 클립 경로와 **동일한 PGM**을 생성합니다.

## 흐름 (과제 공지 그대로)
```
카메라 프레임 → YOLO 검출(digit) → crop → MNIST 정규화 → 28x28 PGM
            → appearance마다 PGM 1장 저장 (spec 파일명)
            → mnistCUDNN가 분류 (INPUT / Result / Total Images / Total Time / per-digit)
```
- 저장 파일명: `frame_<frameidx:06d>_digit_x_conf_<c.cc>_<seq:04d>.pgm` (공지 규약)
- 화면에는 검출 박스 + **현재 보여주는 숫자** + 저장 개수를 실시간 오버레이
- 종료(`q`/`ESC`) 시 저장 폴더 전체에 대해 `mnistCUDNN --dir`을 돌려
  **공식 채점 포맷**을 콘솔에 출력 (이게 채점되는 출력)

## 실행
```bash
# 기본: 자동(appearance) 모드, 카메라 0
python live/live_demo.py

# 수동 캡처: 화면에 박스가 뜨고, SPACE 누른 순간의 프레임만 분류
python live/live_demo.py --manual

# 카메라가 1번이거나 CPU YOLO가 버벅이면
python live/live_demo.py --cam 1 --stride 2 --cam-w 960 --cam-h 540

# 시작 전에 이전 PGM 비우기
python live/live_demo.py --fresh
```

## appearance 분할 = 갭 OR 점프 OR **숫자 내용 변화**
라이브에선 들고 있는 숫자를 드문드문(`--live-every`, 기본 5프레임마다) 분류해서,
**인식 숫자가 바뀌면** 그것도 경계로 잡습니다. 그래서 다음 세 경우 모두 새 숫자가
제대로 저장/인식됩니다:
1. **치우기(갭)** — 숫자를 화면에서 빼면 검출이 끊겨 닫힘
2. **슬라이드(점프)** — 다음 숫자를 옆으로 밀어 넣으면 박스 중심 점프로 닫힘
3. **같은 자리 재작성(내용 변화)** — 지우고 같은 위치에 새로 써도, 인식 숫자가
   바뀌면 닫힘 ← 클립 경로엔 없는 라이브 전용 분할 (이게 "새 숫자 써도 인식 안 됨"
   문제를 해결)

화면의 숫자는 항상 *지금 카메라에 들고 있는* 숫자를 따라갑니다. "3을 지우고 2를 쓰면"
화면도 바로 **2**로 바뀌고(예전엔 닫힐 때만 분류해 한 칸 밀렸음), 슬라이드로 밀어
넣을 때도 이전 숫자가 깜빡이지 않습니다.

> 참고: 분류기 호출(`mnistCUDNN --image`)은 매번 가중치를 다시 로드하므로 매 프레임
> 분류는 무겁습니다. 그래서 `--live-every`로 드문드문 분류합니다(손글씨 추적엔 충분).
> 더 빠른 반응이 필요하면 값을 낮추세요(예 `--live-every 3`). 같은 자리 재작성을 가장
> 깔끔하게 하려면 숫자 사이에 잠깐 화면을 비우는 게 가장 확실합니다.

## 두 가지 시연 모드
- **자동(기본)** — 숫자를 카메라에 잠깐 안정적으로 비추다가 치우면(또는 다음 숫자를
  밀어 넣으면) appearance가 닫히며 PGM 1장 저장 + 즉시 분류. 클립 경로의
  "appearance당 PGM 1장" 규칙과 동일.
- **`--manual`** — 박스는 계속 그려지고, **SPACE**를 누른 순간의 프레임만 캡처/분류.
  시연자가 타이밍을 완전히 제어.

## 주요 옵션
| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--weights` | `../best.pt` | YOLO 가중치 (Orin에서 `.engine` 빌드 시 그 경로 지정) |
| `--bin` | `../mnistCUDNN/mnistCUDNN` | 패치된 분류기 바이너리 |
| `--out` | `../runtime/live_pgm` | PGM 저장 폴더 |
| `--cam` | `0` | 카메라 인덱스 (`/dev/videoN`) |
| `--conf` | `0.35` | YOLO 신뢰도 임계값 |
| `--stride` | `1` | N프레임마다 YOLO 실행 (CPU 지연 시 2~4) |
| `--gap` / `--jump` | `8` / `0.35` | appearance 분할 기준 (갭/점프) |
| `--live-every` | `5` | 들고 있는 숫자 재분류 주기(프레임). 화면 갱신 + 같은자리 재작성 분할 |
| `--manual` | off | SPACE 수동 캡처 모드 |
| `--no-display` | off | 헤드리스(창 없이 콘솔만, 자동 모드) |
| `--fresh` | off | 시작 전 out 폴더 비우기 |
| `--limit` | `0` | 종료 시 `--dir` 분류 개수 제한 (0=전체) |

## 주의
- 이 Orin은 torch에 CUDA가 없어 **YOLO가 CPU로** 돕니다. 고해상도 웹캠은 느릴 수
  있으니 `--cam-w/--cam-h`를 낮추고 `--stride 2`를 쓰세요.
- 분류 정확도/지연/전처리 로직은 본 파이프라인과 100% 공유되므로, 1/3/5 게이트나
  6/8 정확도는 별도로 손댈 필요가 없습니다 — 같은 코드입니다.
